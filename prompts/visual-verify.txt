## Screenshots

**Screenshot directory:** `{{SCREENSHOT_DIR}}`

Read each `.png` file in the screenshot directory using the Read tool.

## Your Review Checklist

For each screenshot, evaluate:

1. **Layout** — Is anything visually broken? Overlapping elements? Misaligned content?
2. **Responsiveness** — Do viewport-specific screenshots show appropriate responsive behavior?
3. **Content** — Is text readable? Are there placeholder/lorem ipsum that shouldn't be there?
4. **Visual Consistency** — Do colors, fonts, spacing match the rest of the app?
5. **Accessibility** — Are there obvious contrast issues? Missing labels?
6. **Regressions** — Does anything look like it regressed from the expected state?

## Output Format

Post a PR comment with your findings:

```markdown
### Visual Verification Report

**Status:** PASS | FAIL | WARN

**Screenshots Reviewed:** [count]

#### Findings

[List any issues found, or "No visual issues detected."]

#### Recommendation

[Proceed to review / Block for visual fixes]

<!-- visual-verify-passed -->  (or <!-- visual-verify-failed -->)
```

If you find **no issues**, use `<!-- visual-verify-passed -->`.
If you find **blocking issues** (broken layout, visual regressions), use `<!-- visual-verify-failed -->`.

## Important

- You are reviewing the VISUAL output only. Do not review code.
- Focus on what the USER would see, not implementation details.
- Be concise. Only flag genuine visual issues, not style preferences.

When done, run `/exit`.
